Package org.springframework.core.type

Interface AnnotatedTypeMetadata

  • All Implemented Interfaces:
    
public interface AnnotatedTypeMetadata

    Defines access to the annotations of a specific type (class or method), in a form that does not necessarily require class loading of the types being inspected. Note, however, that classes for encountered annotations will be loaded.

      • getMergedRepeatableAnnotationAttributes

         Set<AnnotationAttributes> getMergedRepeatableAnnotationAttributes(Class<out Annotation> annotationType, Class<out Annotation> containerType, boolean classValuesAsString)

        Retrieve all repeatable annotations of the given type within the annotation hierarchy above the underlying element (as direct annotation or meta-annotation); and for each annotation found, merge that annotation's attributes with matching attributes from annotations in lower levels of the annotation hierarchy and store the results in an instance of AnnotationAttributes.

        @AliasFor semantics are fully supported, both within a single annotation and within annotation hierarchies.

        Parameters:
        annotationType - the annotation type to find
        containerType - the type of the container that holds the annotations
        classValuesAsString - whether to convert class references to String class names for exposure as values in the returned AnnotationAttributes,instead of Class references which might potentially have to be loadedfirst

      • getMergedRepeatableAnnotationAttributes

         Set<AnnotationAttributes> getMergedRepeatableAnnotationAttributes(Class<out Annotation> annotationType, Class<out Annotation> containerType, boolean classValuesAsString, boolean sortByReversedMetaDistance)

        Retrieve all repeatable annotations of the given type within the annotation hierarchy above the underlying element (as direct annotation or meta-annotation); and for each annotation found, merge that annotation's attributes with matching attributes from annotations in lower levels of the annotation hierarchy and store the results in an instance of AnnotationAttributes.

        @AliasFor semantics are fully supported, both within a single annotation and within annotation hierarchies.

        If the sortByReversedMetaDistance flag is set to true, the results will be sorted in reversed order based on each annotation's meta distance, which effectively orders meta-annotations before annotations that are declared directly on the underlying element.

        Parameters:
        annotationType - the annotation type to find
        containerType - the type of the container that holds the annotations
        classValuesAsString - whether to convert class references to String class names for exposure as values in the returned AnnotationAttributes,instead of Class references which might potentially have to be loadedfirst
        sortByReversedMetaDistance - true if the results should besorted in reversed order based on each annotation's meta distance
